## Cursor 快速上手指南

### 1. 先搞清楚两种模式

- **Ask 模式（只读）**：我只回答、解释、给建议，不会改文件、运行命令。适合：问问题、学概念、看代码。
- **Agent 模式（可执行）**：我可以改代码、运行命令、创建/修改文件。适合：实现功能、修 bug、做小项目。

想“玩转” Cursor，就要在**需要动手改代码时用 Agent 模式**。

### 2. 必会的基础操作

| 操作 | 说明 |
|------|------|
| **Cmd + K**（Mac）或 **Ctrl + K**（Win） | 在光标处唤起“内联编辑”：选中代码，描述你想怎么改，我会直接改这一段。 |
| **Cmd + L**（Mac）或 **Ctrl + L**（Win） | 打开 AI 聊天面板，可以问问题、给任务、贴代码。 |
| **@ 引用** | 在输入框里输入 `@`，可以引用文件、文件夹、文档、规则等，让我更精准地理解上下文。 |
| **Cmd + I**（Mac）或 **Ctrl + I**（Win） | 打开 Composer，适合做“多文件、多步骤”的复杂任务。 |

先把 **Cmd/Ctrl + K** 和 **Cmd/Ctrl + L** 用熟，就够应付大部分日常需求。

### 3. 如何“玩转” Cursor：实用习惯

- **说清楚你要什么**  
  例如：“在这段函数里加错误处理”“用 React 写一个可复用的 Button 组件”。越具体，结果越好。

- **多用 @ 引用**  
  问问题时可以 `@文件名` 或 `@文件夹`，让我基于你的项目回答，而不是泛泛而谈。

- **分步骤做复杂功能**  
  大功能拆成：先写接口/组件结构 → 再实现逻辑 → 最后加样式/测试。每步都可以用一次 Cmd+K 或 Composer。

- **看我的修改再决定**  
  我改的代码会以 diff 形式出现，你可以接受、拒绝或自己再改，养成“先看再接受”的习惯。

- **用规则和技能（可选进阶）**  
  - 在项目里加 `.cursor/rules` 或 `AGENTS.md`，可以定制“这个项目里我希望 AI 怎么做事”。  
  - 你还可以创建自己的 Skill，让我按固定流程或风格干活。

### 4. 新手推荐练习路径

1. **今天**：用 **Cmd + L** 问我一个你正在写的项目里的小问题（例如“这段代码在干什么？”）。
2. **明天**：选一小段代码，用 **Cmd + K** 让我“加注释”或“改成更清晰的写法”，感受内联编辑。
3. **本周**：用 **Composer（Cmd + I）** 提一个完整小需求，比如“在这个项目里加一个 XXX 页面/功能”，让我从零写或改多文件。
4. **之后**：尝试用 `@` 引用文件、用 Agent 模式让我直接改代码和运行命令。
