可以这样写：在 `if` 里集中做「不合理就 return」的校验，兼顾数字输入框里 e、E、小数点等。

```javascript
function onPhoneChange(value) {
    const str = String(value)
    // 科学计数法符号，数字输入框允许但电话号码不需要
    if (/[eE]/.test(str)) return
    // 小数点，电话号为整数
    if (str.includes('.')) return
    // 只允许数字、可选的前导 +、以及常见分隔符 - 空格 ()
    if (/[^0-9+\-\s()]/.test(str)) return
    // 避免 NaN 等无效数值（例如单独输入 + - 或多个符号）
    if (str.trim() !== '' && !/^[\s+\-()]*[0-9][0-9\s+\-()]*$/.test(str.trim())) return

    const [_, dispatch] = max.globalFn.useReducer(max.localFn.rootReducer())
    const customerInfo = max.data.get('customerInfo')
    dispatch({ type: "UPDATE_CUSTOMER_INFO", customerInfo: { ...customerInfo, phone: value } })
}
```

**各条校验在防什么：**

| 校验 | 场景 |
|------|------|
| `/[eE]/` | 数字框里合法的 `e`/`E`（科学计数法），电话里不允许 |
| `str.includes('.')` | 小数点，电话号应为整数 |
| `/[^0-9+\-\s()]/` | 除数字、`+`、`-`、空格、括号外的字符（如其它字母、逗号等） |
| 最后一个 `test` | 至少包含一个数字，且整体是「可接受的电话号格式」，避免只有 `+`、`-`、空格等时也通过 |

如果你们电话号**只允许纯数字**（不要 `+`、空格、括号等），可以改成只允许数字：

```javascript
if (/[^0-9]/.test(str)) return
```

并在前面保留对 `e`、`E` 和 `.` 的检查（或直接用 `/[^0-9]/` 一并拦住）。
